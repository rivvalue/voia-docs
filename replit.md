# Overview
VOÏA (Voice Of Client) is a Flask-based system for comprehensive customer feedback collection and AI-powered analysis, specializing in Net Promoter Score (NPS) surveys. Its purpose is to convert raw customer feedback into actionable insights, identifying sentiment, key themes, churn risk, and growth opportunities. VOÏA provides businesses with a robust tool for understanding customer sentiment, improving services, and fostering organic growth through AI-driven analysis of customer interactions. The project features a production-ready multi-tenant participant management system with extensive email delivery capabilities, AI-powered conversational surveys with hybrid prompt architecture, and participant segmentation for personalized experiences and advanced analytics.

**Language Support**: Full French localization implemented (October 2025) with 2,102 translated strings across UI templates, flash messages, and email templates. Translation uses manual ChatGPT workflow with JSON format for fastest time-to-market.

# User Preferences
Preferred communication style: Simple, everyday language.
User interface tone: Thought leadership and research-oriented language, avoiding sales-oriented messaging.
Project customization: Rivvalue Inc. branding and conversational AI surveys for enhanced user experience.
Target market: French-speaking businesses and organizations.

# System Architecture
The system is a Flask web application with a multi-tiered architecture. The frontend uses Jinja2, Bootstrap 5, custom CSS, vanilla JavaScript, and Chart.js. The backend is Flask with SQLAlchemy ORM, designed for scalability from SQLite to PostgreSQL. AI integration, primarily via OpenAI API, handles natural language processing, sentiment analysis, and conversational surveys (VOÏA), supplemented by TextBlob.

**UI/UX**: Multi-step survey forms, interactive dashboards, chat-style interfaces for conversational surveys, and Rivvalue Inc. branding with a professional blue color scheme. Modern sidebar navigation, mobile responsiveness, and active state highlighting. The Settings Hub v2 features a 5-card accordion layout, providing access to Account, User, Data, System, and Platform Administration settings (role-restricted). Platform Administration offers cross-tenant tools like Business Analytics Hub, Onboarding, License Dashboard, and Platform Admin Users. Modernized dashboard and campaign pages feature consistent typography, clean white backgrounds with red accents, lighter shadows, and standardized badge styling. The home page has a minimalist design with red accent borders for features and optimized mobile layouts. Business Intelligence Navigation has a three-tier structure: Overview, Executive Summary, and Campaign Insights. Session-based Breadcrumb Navigation ensures intelligent back-button behavior. Admin pages and the Survey Response page are redesigned with Settings Hub v2 patterns, featuring bold red gradient headers, VOÏA brand color tokens, comprehensive ARIA accessibility attributes, and mobile-responsive breakpoints. All inline styles are migrated to custom.css, and emoji icons are replaced with FontAwesome. A comprehensive breadcrumb system is implemented across business pages using a reusable component. **Demo platform pages (index.html, demo_intro.html, server_auth.html, conversational_survey.html) are fully V2 compliant:** zero inline styles, CSS variable tokens, WCAG 2.1 AA accessibility compliance (comprehensive ARIA landmarks, semantic HTML5, keyboard navigation), standardized V2 card/button/typography patterns, JavaScript refactored to use CSS classes (hideElement/showElement helpers) for all state transitions, and mobile-responsive breakpoints. (Completed October 2025)

**Technical Implementations**:
-   **Survey Collection**: Multi-step forms with dynamic follow-up questions and real-time validation.
-   **AI Analysis Engine**: Sentiment analysis, key theme extraction, churn risk assessment, growth opportunity identification, and NPS-based growth factor analysis.
-   **Conversational Surveys (VOÏA)**: AI-powered (GPT-4o) natural language interface with advanced personalization capabilities, featuring a hybrid prompt architecture combining structured JSON with natural language guidance for dynamic question generation and structured data extraction. Includes participant segmentation and feature flag rollout capabilities.
-   **Data Management**: Centralized data aggregation, NPS calculation, time-based filtering, and optimized database queries.
-   **Authentication**: JWT token-based with email validation, admin roles, server-side token generation, and automatic invalidation.
-   **Performance**: PostgreSQL-backed persistent task queue (DEPLOYED - zero data loss on restarts), database indexing, connection pooling, asynchronous background tasks for AI, IP-based rate limiting, optimized dashboard queries, admin-configurable response caching, index page response caching, Executive Summary comparison optimization, frontend optimizations, and LicenseService query optimization.
-   **Audit Trail System**: Comprehensive audit logging with accurate timestamp preservation. Captures action timestamps immediately at occurrence (before any queue delays) and passes them through async processing to ensure audit logs reflect actual action time, not database insertion time. PostgreSQL task queue integration via Flask request context (g.task_queue) enables async processing of all 51 system audit points. (Deployed October 2025)
-   **Security**: Token-based authentication, duplicate response prevention, enhanced rate limiting, robust input validation, CSRF/XSS protection, and Sentry integration.
-   **Branding**: "VOÏA - Voice Of Client" with "AI Powered Client Insights" subtitle, specific tagline, multi-tenant logo system, and selective trial branding.
-   **Multi-Tenant Architecture**: Business Accounts, Campaigns, and Participants with tenant isolation via `business_account_id` scoping, dual authentication, and a token system for survey access, including a lightweight scheduler. Participant entities support optional segmentation attributes (role, region, customer_tier, language, tenure_years) for personalized survey experiences and advanced analytics segmentation.
-   **Participant Management Enhancements**: Individual participant editing with conditional email locking (locked when survey history exists), deletion protection for participants with survey responses, tenure tracking (tenure_years), and comprehensive audit logging for all participant changes. Bulk edit functionality allows updating up to 50 participants simultaneously for role, region, customer_tier (locked for participants with survey history), language, tenure, and commercial_value. Bulk operations generate unique bulk_operation_id for audit trail correlation. Company_name excluded from bulk edits to maintain data integrity. Advanced multi-value filtering for campaign participant assignment with indexed columns (company_name, role, region, customer_tier, language, tenure_years), supporting multiple selection per attribute with AND/OR semantics, NULL value handling as "Unspecified", expanded-by-default V2-compliant filter panel with Select2 searchable multi-selects, persistent filter summary bar with removable chips, "Showing X of Y" result count feedback, and intelligent empty-state messaging. (Deployed October 2025)
-   **Manual Commercial Value Tracking**: Company-level commercial value system for accurate account intelligence, stored on the Participant model, with CSV upload validation and automatic synchronization across participants from the same company.
-   **Email Delivery System**: Multi-provider email infrastructure (AWS SES, SMTP) with business accounts choosing providers, encrypted password storage, connection testing, professional VOÏA-branded templates, background task processing, delivery tracking, and configurable email content at both business account and campaign levels, with a 3-tier fallback architecture for customization. Includes an automated reminder system with campaign-level configuration (toggle, delay), SQL-optimized batch processing, EmailDelivery tracking, and reminder metrics (sent count, conversion rate).
-   **Campaign Lifecycle Management**: Automated status transitions, multi-tenant scheduling, automatic KPI snapshot generation, and background task management with comprehensive audit logging. Draft campaigns support full editability and deletion with cascade removal of participant associations, protected by status validation and multi-tenant security. Manual lifecycle control preserves user agency with explicit transitions (draft → ready → active → completed). All campaign status changes are logged to the audit trail.
-   **Participant Status Reconciliation**: Automated nightly reconciliation system that ensures data consistency between email/response tracking (engagement_metrics) and participant lifecycle status (participant_stats). Trial users use atomic transaction pattern (status update before commit), invited users use mark_survey_completed() helper with auto_commit=False for atomicity. Background reconciliation job runs every 24 hours via PostgreSQL task queue scheduler, processing each drift correction in isolated transactions with rollback protection. Campaign view page includes tooltips explaining metric differences and relabeled "Responses Received" for clarity. (Deployed October 2025)
-   **Hybrid Survey Customization**: Campaign-specific survey personalization with business account defaults.
-   **License Management System**: Enterprise-ready license management with usage tracking and enforcement.
-   **Business Account User Management**: Multi-tenant user management with professional UI, license-aware counters, and comprehensive user workflows.
-   **Mandatory Onboarding System**: Extensible guided setup workflow for business account administrators.
-   **Feature Flag System**: Production-ready infrastructure for UI version toggling, including a PostgreSQL-backed task queue.
-   **Segmentation Analytics**: Comprehensive analytics system displaying NPS and satisfaction metrics segmented by participant attributes (role, region, customer tier). Includes a dedicated Segmentation Insights tab in Campaign Insights with interactive Chart.js visualizations showing NPS distribution, detailed satisfaction ratings table, and historical preservation through campaign snapshots. Supports NULL handling by grouping unmapped participants as "Unspecified".
-   **Demo Data Generation**: High-volume test data scripts capable of generating realistic test data for development and staging, populating all participant segmentation fields and enforcing commercial value consistency.
